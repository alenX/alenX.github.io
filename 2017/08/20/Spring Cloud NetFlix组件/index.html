Spring Cloud NetFlix组件

发表于 2017-08-20

Spring Cloud NetFlix

提供的模块包括Eureka(服务发现)、Hystrix(断路器)、Zuul(智能路由)、Ribbon(客户端负载均衡)。

服务发现Eureka

  • 服务发现是基于微服务架构的关键原则之一。每个服务器将注册服务的状态复制到其他服务器。
  • 如果客户端指定心跳时间内不可达，那么服务端将该服务实例移除
  • @EnableDiscoveryClient和@EnableEurekaClient的区别，在只有一个Eureka服务器时可以使用@EnableDiscoveryClient
  • eureka.instance配置实例相关信息
  • the instance, the server and the client 三部分元数据统一，每个心跳时间为30s，所以需要全部同步，需要90s。eureka.instance.leaseRenewalIntervalInSeconds属性可以设置该时间
  • Eureka区块的概念？
  • 结点复制以及集群实现，spring.profiles=master
    eureka.instance.hostname=master
    eureka.client.serviceUrl.defaultZone=http://slaves:8762/eureka
  • Region（区域默认us-east-1）以及Zone（可用区us-east-1c）的概念，Spring Cloud中默认的region是us-east-1 。

服务发现的独立模式 ，关闭客户端行为，不会继续尝试并且无法访问对等体。

Zuul

  • EnableZuulProxy 和EnableZuulServer的区别，过滤器的不同，建议使用EnableZuulProxy
  • ZuulFilter的有效范围，路由控制的全部功能均受该Filter控制

  • 路由映射，可以使用正则匹配方式配置

  • 可以配置忽略路由地址信息 zuul.ignoredPatterns: //admin/

验证与安全保障: 识别面向各类资源的验证要求并拒绝那些与要求不符的请求。

审查与监控: 在边缘位置追踪有意义数据及统计结果，从而为我们带来准确的生产状态结论。

动态路由: 以动态方式根据需要将请求路由至不同后端集群处。

压力测试: 逐渐增加指向集群的负载流量，从而计算性能水平。

负载分配: 为每一种负载类型分配对应容量，并弃用超出限定值的请求。

静态响应处理: 在边缘位置直接建立部分响应，从而避免其流入内部集群。

Netflix公司还利用Zuul的功能通过金丝雀版本实现精确路由与压力测试。

虽然提供的功能还算丰富，但都比较弱，很难满足高要求的场景。
